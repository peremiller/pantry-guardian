# Contributing

1. Create a focused branch and keep changes scoped.
2. Install with `npm ci` and run `npm run check` plus `npm test` before submitting.
3. Never commit credentials, personal data, generated build output, or local pantry data.
4. Add server-side validation for every public or AI endpoint and preserve the food-safety guards.
5. Describe behavior changes, validation performed, accessibility impact, and screenshots for visual changes.

Use clear commit messages, prefer typed helpers over duplicated logic, and keep user-facing errors actionable without exposing internal details.
